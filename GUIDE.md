✦ 若您目前尚未打算將專案安裝至 /opt 或掛載 systemd，僅想在當前開發目錄下進行手動運作與測試，請參考以下步驟：

  1. 準備環境
  確保您的 Raspberry Pi 已安裝必要的套件：
   1 sudo apt update
   2 sudo apt install python3-rpi.gpio chrony -y


  2. 硬體接線
  請依照我們先前優化的建議接線（若您尚未接線）：
   * VCC -> Pin 1 (3.3V)
   * GND -> Pin 20 (GND)
   * CLK -> GPIO 23 (Pin 16)
   * DAT -> GPIO 24 (Pin 18)
   * RST -> GPIO 25 (Pin 22)

  3. 手動測試與操作流程


  由於涉及 GPIO 操作與系統時間設定，所有指令皆需使用 sudo。

  A. 硬體功能驗證 (最重要)
  執行我們建立的測試腳本，它會執行「讀取 -> 寫入當前時間 -> 再次讀取並比對」的完整流程。


   1 sudo python3 ds1302_test.py
   * 成功指標：看到 SUCCESS: RTC write verified 字樣。

  B. 模擬開機恢復時間 (RTC -> 系統)
  如果您想手動觸發「從 RTC 讀取並設定系統時間」的動作：


   1 sudo python3 rtc_init.py
   * 驗證方式：執行 date 指令查看系統時間是否已變更。
   * 測試技巧：您可以先用 sudo date -s "2022-01-01" 把系統時間調亂，再執行 rtc_init.py 看它是否恢復正確時間。

  C. 手動同步時間至硬體 (系統 -> RTC)
  當您確定系統時間正確（例如已連網），想把它寫入 RTC 備份時：


   1 sudo python3 rtc_sync.py
   * 注意：此腳本預設會檢查 chronyc tracking（NTP 同步狀態）。如果您的 Pi 沒連網，它可能會跳過更新。
   * 強制更新技巧：若想不理會 NTP 狀態強制寫入，可直接使用 ds1302_test.py 或者稍微修改 rtc_sync.py。

  D. 即時查看日誌
  專案在執行時會將詳細過程記錄在本地的 logs/ 目錄下：


   1 tail -f logs/rpi-rtc-manager.log

  ---

  4. 開發建議：如何調整組態？
  如果您想更換腳位，不需修改程式碼，直接編輯目錄下的 config/rtc.conf：


   1 CLK=23
   2 DAT=24
   3 RST=25
  程式執行時會自動偵測並載入這些數值。


  5. 常見測試情境
   1. 冷啟動測試：手動執行 rtc_init.py 後，拔掉網路線重啟（不掛載 service），再次手動執行 rtc_init.py，確認時間是否連續。
   2. 異常測試：故意拔掉一顆 GPIO 線，執行 ds1302_test.py，確認程式是否會噴出錯誤日誌而非造成系統當機（這驗證了我們剛才做的防阻塞機制）。


  透過這種方式，您可以完全掌握專案在進入自動化階段前的所有行為。
